# クラス図

テーブルオーダーシステムの主要なクラス構成を示す図です。
特にDAO層のインターフェース抽象化と、サービス層・コントローラー層との関係を視覚化しています。

```mermaid
classDiagram
    %% 共通インターフェース
    class BaseDAO~T~ {
        <<interface>>
        +findAll() List~T~
    }

    %% DAOインターフェース
    class UserDAO {
        <<interface>>
        +login(String id, String password) User
    }
    class CategoryDAO {
        <<interface>>
        +insert(String name) boolean
    }
    class ProductDAO {
        <<interface>>
        +findByCategory(int categoryId) List~Product~
        +findById(int id) Product
        +insert(Product p) boolean
        +update(Product p) boolean
        +updateAvailability(int productId, boolean isAvailable) boolean
    }
    class OrderDAO {
        <<interface>>
        +insertOrder(Connection con, int tableId, int status) int
        +insertOrderItems(Connection con, int orderId, List~CartItem~ items, int status) void
        +findActiveOrderItems() List~OrderItemView~
        +updateItemStatus(int itemId, int status) boolean
        +findUnsettledTables() List~TableOrderSummary~
        +getTableOrderSummary(int tableId) TableOrderSummary
        +findAllTableStatus() List~TableStatusView~
        +getTotalSales() int
        +findDailySales() List~DailySales~
        +findProductSalesRanking() List~ProductSales~
    }

    %% 継承関係
    BaseDAO <|-- UserDAO
    BaseDAO <|-- CategoryDAO
    BaseDAO <|-- ProductDAO

    %% 実装クラス
    class UserDAOImpl {
        -upgradeToBcrypt(String userId, String hash)
    }
    class CategoryDAOImpl
    class ProductDAOImpl
    class OrderDAOImpl

    UserDAO <|.. UserDAOImpl
    CategoryDAO <|.. CategoryDAOImpl
    ProductDAO <|.. ProductDAOImpl
    OrderDAO <|.. OrderDAOImpl

    %% サービス層
    class OrderService {
        -OrderDAO orderDAO
        +createOrder(int tableId, List~CartItem~ items) boolean
        +completeCheckout(int tableId) boolean
    }
    OrderService --> OrderDAO : uses

    %% データベース管理
    class DBManager {
        +getConnection() Connection
        +getPepper() String
        +closeConnection(Connection con)
        +closePool()
    }

    %% 関連
    UserDAOImpl ..> DBManager : uses
    CategoryDAOImpl ..> DBManager : uses
    ProductDAOImpl ..> DBManager : uses
    OrderDAOImpl ..> DBManager : uses

    %% ノート
    note for BaseDAO "Generic interface for common CRUD"
    note for OrderService "Transaction management"
```

## 各層の役割

- **Controller層 (Servlets)**: リクエストを受け取り、適切なServiceやDAOを呼び出して結果をJSPへ渡します。
- **Service層 (OrderService)**: 複数のDAO呼び出しを組み合わせた複雑なビジネスロジックや、トランザクション境界（commit/rollback）を管理します。
- **DAO層 (Data Access Objects)**: データベースとの直接的な対話を受け持ちます。インターフェースによって具象実装が隠蔽されています。
- **Database (DBManager)**: 接続プール（HikariCP）の管理とコネクションの提供を行います。
