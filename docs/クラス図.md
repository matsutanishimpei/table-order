# クラス図

テーブルオーダーシステムの主要なクラス構成を示す図です。
最新のアーキテクチャでは、Service層の全面導入、モデルの `record` 化、および依存性注入（DI）のための `ServiceFactory` が導入されています。

```mermaid
classDiagram
    %% Service Factory
    class ServiceFactory {
        +getCategoryService() CategoryService
        +getProductService() ProductService
        +getOrderService() OrderService
        +getTableService() TableService
        +getUserService() UserService
        +getSalesService() SalesService
    }

    %% Service層
    class CategoryService { <<interface>> }
    class ProductService { <<interface>> }
    class OrderService {
        <<interface>>
        +createOrder(int tableId, List~CartItem~ items) boolean
        +completeCheckout(int tableId) boolean
    }
    class TableService { <<interface>> }
    class UserService { <<interface>> }
    class SalesService { <<interface>> }

    ServiceFactory ..> CategoryService : creates
    ServiceFactory ..> ProductService : creates
    ServiceFactory ..> OrderService : creates
    ServiceFactory ..> TableService : creates
    ServiceFactory ..> UserService : creates
    ServiceFactory ..> SalesService : creates

    %% DAO層
    class CategoryDAO { <<interface>> }
    class ProductDAO { <<interface>> }
    class OrderDAO {
        <<interface>>
        +insertOrder(Connection con, int tableId, int status) int
        +insertOrderItems(Connection con, int orderId, List~CartItem~ items, int status) void
        +findActiveOrderItems() List~OrderItemView~
        +updateItemStatus(int itemId, int status) boolean
    }
    class TableDAO {
        <<interface>>
        +findUnsettledTables() List~TableOrderSummary~
        +getTableOrderSummary(int tableId) Optional~TableOrderSummary~
        +findAllTableStatus() List~TableStatusView~
    }
    class SalesDAO {
        <<interface>>
        +getTotalSales() int
        +findDailySales() List~DailySales~
        +findProductSalesRanking() List~ProductSales~
    }

    OrderServiceImpl --> OrderDAO : uses
    TableServiceImpl --> TableDAO : uses
    SalesServiceImpl --> SalesDAO : uses

    %% 外部連携・ユーティリティ
    class ImageStorageProvider {
        <<interface>>
        +upload(InputStream is, String fileName) String
        +delete(String identifier) boolean
    }
    class CloudinaryUtil {
        -Cloudinary cloudinary
    }
    ImageStorageProvider <|.. CloudinaryUtil

    %% 例外階層
    class BaseException { <<abstract>> }
    class BusinessException
    class SystemException
    class DatabaseException
    BaseException <|-- BusinessException
    BaseException <|-- SystemException
    SystemException <|-- DatabaseException

    %% モデル (record)
    class User { <<record>> }
    class Product { <<record>> }
    class OrderItemView { <<record>> }
    class TableOrderSummary { <<record>> }

    %% データベース管理
    class DBManager {
        +getConnection() Connection
        +getPepper() String
    }

    %% 関連
    DAO ..> DBManager : uses
    DAO ..> SqlConstants : uses SQL constants
```

## 各層の役割

- **Controller層 (Servlets)**: `ServiceFactory` を介してサービスを取得し、ビジネスロジックを実行します。入出力の制御に専念します。
- **Service層**: 業務ルール、トランザクション管理、外部サービス連携を担当します。インターフェースによって抽象化されています。
- **DAO層**: データベース操作を担当します。SQLは `SqlConstants` に集約され、静的な安全性が確保されています。
- **Model層**: `record` を使用して不変なデータモデルを定義します。
- **Exception層**: 業務エラーとシステムエラーを階層的に管理し、一貫したエラーハンドリングを実現します。
- **Utility層**: 画像アップロード（Cloudinary）やパスワードハッシュ化などの共通機能を提供します。
