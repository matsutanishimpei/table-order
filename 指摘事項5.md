# 指摘事項5: Findyスキル偏差値向上のためのリファクタリング提案

Findy などのコード解析プラットフォームで高い評価を得るために、モダンな Java プログラミング（Java 17基準）とクリーンアーキテクチャの原則に基づいたリファクタリング案を提案します。

## 1. モダンな Java 機能の活用 (Modern Java)

### 1.1 Optional による Null 安全な設計 【完了】
- **現状**: `DAO` や `Service` の `findById` メソッドが、データが存在しない場合に `null` を返しています。
- **リファクタリング案**: 戻り値の型を `Optional<Product>` に変更します。
- **効果**: 呼び出し側に `isPresent()` や `orElseThrow()` の使用を強制することで、`NullPointerException` を防ぎ、意図を明確にします。Findy では「モダンな言語機能の活用」として高く評価されます。

### 1.2 ストリーム API による宣言的なリスト処理 【完了】
- **現状**: `for` ループや `while` ループによるリストの加工。
- **リファクタリング案**: `List<Product>` のフィルタリングや変換に `Stream API` を積極的に活用します。
  - 例：`products.stream().filter(Product::isAvailable).collect(Collectors.toList());`
- **効果**: コードの宣言的な記述が可能になり、可読性が向上します。

## 2. アーキテクチャとデザインパターン (Design Patterns)

### 2.1 共通例外ハンドリングの導入 【完了】
- **現状**: 各サーブレットで個別に `try-catch` や `throw` を行っています。
- **リファクタリング案**: `BaseServlet` を作成するか、サーブレットフィルタを活用して共通の例外ハンドリングを実装します。
- **効果**: ビジネスロジックからボイラープレートなエラーハンドリングを分離でき、「関心の分離」が評価されます。

### 2.2 バリデーションの高度化
- **現状**: `ValidationUtil` での boolean による単純な検証。
- **リファクタリング案**: バリデーション結果を「成功/失敗+エラーメッセージ」を持つ `ValidationResult` オブジェクトで返すようにします。
- **効果**: ユーザーへのフィードバックが容易になり、検証ロジックの再利用性が高まります。

## 3. コードの保守性と可読性 (Maintainability)

### 3.1 定数管理の徹底
- **現状**: SQL クエリや JSP のパス、バリデーションルール（許可する画像拡張子など）がメソッド内にハードコードされています。
- **リファクタリング案**: これらを `final static` 定数、あるいは設定ファイル（`.properties`）に切り出します。
  - 例：`ValidationUtil` 内の `allowedTypes` を `private static final Set<String> ALLOWED_TYPES` として定義。
- **効果**: 定義箇所が一箇所になり、変更に強いコードになります。

### 3.2 SLF4J による適切なログ出力 【完了】
- **現状**: ログ出力が不足しているか、一部で標準出力（`e.printStackTrace()`）が使用されている可能性があります。
- **リファクタリング案**: すべてのレイヤーで `slf4j` を使用し、ログレベル（INFO, WARN, ERROR）を適切に使い分けます。
- **効果**: エンタープライズ品質のコードとして評価され、デバッグの効率も上がります。

## 4. テストコードの品質 (Testing Excellence)

### 4.1 JUnit 5 の機能をフル活用 【完了】
- **現状**: 基本的な単体テストの実装。
- **リファクタリング案**: `@ParameterizedTest` を活用して、バリデーションの境界値テスト（空文字、最大文字数、不正フォーマットなど）を効率的に網羅します。
- **効果**: 少ないコード量で高い網羅率を実現でき、テスト設計能力の高さが示せます。

### 4.2 DB テストのポータビリティ 【完了】
- **現状**: ローカル DB に依存した統合テスト。
- **リファクタリング案**: `Testcontainers` の導入（環境が許せば）や、H2 Database などのインメモリ DB を活用した、環境に依存しないテスト実行環境を構築します。
- **効果**: CI/CD との親和性が極めて高く、DevOps 観点での評価が上がります。
